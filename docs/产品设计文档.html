PART 01 · 概要设计

产品概要设计

本部分从顶层定义「AI 小博士」是什么、为谁解决什么问题、凭什么解决，以及产品的整体功能骨架与设计哲学。它是后续详细设计与交互规格的总纲。

1.1产品定位

AI 小博士是一款面向中小学一线教师的 AI 备课与教学创作助手。它把教师日常「找资源、出卷子、做课件、写教案、问教材、画导图」等高频任务，统一收敛到一个智能入口里：教师用自然语言说出需求，AI 自动识别意图、匹配场景，并基于学科网权威资源库完成创作。

与通用大模型最本质的区别在于一句话：小博士的每一份产出都站在学科网资源库之上 —— 不是凭空生成，而是有据可依。

一句话定位 懂教学、有出处、越用越懂你 —— 一个建立在学科网 20 年教研沉淀之上的 AI 教学助手。

设计目标

1

降门槛

不需要学习复杂功能菜单，一句话即可启动任意创作任务。

2

可信赖

内容有权威资源库背书、可溯源到教材原文与章节页码。

3

懂个人

记住教师的学科、学段、版本与偏好，越用越省心。

1.2目标用户与核心场景

核心用户是中小学（K-12）各学科一线教师，典型画像如文档中的「李老师 —— 初中数学教师，常用人教版、偏好中等难度」。教师的真实工作流被拆解为以下高频需求，并一一对应到产品场景：

场景的划分并非拍脑袋，而是建立在对 5 月全量真实提问数据（60,802 行）的抽样分析之上 —— 形成了 13 大类、23 个末级分类 的问题分类体系（详见 2.2 节）。

1.3价值主张：权威底座

「权威」是小博士贯穿全局的信任锚点。无论生成还是查找，每一份产出背后都有学科网资源库支撑，并在界面上以「检索学科网权威资源库 → 匹配三审三校精品内容作为创作底座」的步骤被显性传达给教师。

1.5核心设计理念

①

对话即入口

不强迫教师先选功能。说出需求即可，AI 判断该进哪个场景；判断不了就直接回答。降低「选择成本」。

②

有据可依

生成有资源库底座，查找有出处，答疑可溯源。把「可信」做成可被看见的产品语言。

③

一条会话贯穿

跨场景切换时，左侧对话线索不断 —— 教师在「同一位助手」面前连续工作，而非在割裂的工具间跳转。

④

越用越懂你

从创作与收藏中学习偏好，预填表单、提供续作，记忆透明可管理、可关闭。

1.6关键流程概览

无论从哪个入口进入，主干流程都收敛为下面五步。这是理解整个产品交互的「主旋律」。

1

表达需求

首页输入框写下一句话，或直接点击某个场景快捷入口 / 示例。

2

意图识别

AI 在对话气泡内逐步展示：理解需求 → 提取关键信息 → 检索权威库 → 匹配场景（带匹配度）。

3

进入工作台

带着识别结果落入对应场景的「左对话 + 右产出」分栏工作区。

4

多轮迭代

在对话里持续追加要求（换题、加页、改难度…），每一轮产出被冻结为可回看的「轮次」。

5

沉淀复用

产出存入「我的内容」、资源进「资源篮」、偏好进「记忆」，下次可一键续作。

PART 02 · 详细设计

产品详细设计

本部分逐一拆解产品的信息架构、意图识别引擎、七大场景工作台、贯穿式会话模型、记忆系统、账号权限与全局沉淀模块，并给出视觉设计系统规范。

2.1信息架构与导航

整个应用由两类「外壳」构成：首页外壳（首页 / 记忆 / 内容 / 历史等沉淀页）与工作台外壳（七大场景）。两者共享同一条左侧常驻导航栏（LeftRail），确保菜单在任何界面都不消失。

左侧导航栏（LeftRail）

导航栏支持展开（252px）/ 收起（72px）两态，状态持久化于本地；移动端转为可呼出的抽屉。其结构自上而下为：

  • 品牌区 — 小博士头像 + 名称；收起态下悬停切换为「展开」按钮。
  • 新对话 — 主操作按钮（品牌渐变），清空会话回到首页。
  • 主导航 — 我的记忆 · 我的内容 · 资源篮（带数量角标）· 历史对话。
  • 历史对话列表 — 展开态下显示近期会话，点击可恢复整段对话。
  • 账号区（底部）— 登录态显示头像与账号菜单；未登录显示登录入口。

页面地图

外壳页面 / 视图说明登录要求

设计原则「过程」与「结果」分离：历史对话沉淀的是会话过程（可续聊），我的内容沉淀的是最终产物（可复用），两者互为补充。

2.2意图识别与问题分类体系

意图识别是产品的「大脑」。它决定一句话该进入哪个工作台、还是留在通用助手回答。识别逻辑建立在对真实数据的分类研究之上。

分类体系：13 大类 / 23 末级分类

以「资源 / 输出类型」为第一维度组织大类，再以「查找 vs 生成」为二级维度拆分子类。颗粒度优先，不强行合并不同意图。

识别四步法与显性化

识别过程不是黑盒，而是被设计成对话气泡内逐步点亮的可见步骤，让教师「看见 AI 在想什么」：

1

理解需求

原句回显，确认 AI 收到的是什么。

2

提取关键信息

抽取实体并以标签呈现：版本 / 学段 / 学科 / 知识点（如「人教版 · 七年级 · 数学 · 有理数」）。

3

检索学科网权威库

匹配三审三校精品内容作为创作底座（信任锚点）。

4

匹配场景

给出目标场景卡片 + 匹配度（如 97%）；未命中专用工具则交由通用助手直接回答。

路由判定规则（按优先级）

识别引擎遵循「命中即停」的判断树，关键规则如下（实现于关键词 + 结构特征匹配）：

  • 优先排噪 — 纯网址、单字符/无意义、脱离上下文的续问片段，先行拦截或归入闲聊。
  • 资源类型优先 — 命中「教案/课件/试卷/讲义/导图」等内容类型词，再以是否含生成动词区分查找(a)与生成(b)。
  • 咨询/规划前置 — 「公开课建议 / 教学计划 / 复习方案」等先于制作工具，路由到通用助手，避免被强行拉进某个生成工具。
  • 数据驱动分析 — 含「平均分/及格率/优秀率」且带数字 → 成绩分析（通用助手）。
  • 兜底 — 无法归入任何专用场景 → 通用助手直接对话回答。
工作台内的「弱切换」保护在某个工作台内迭代时，采用更严格的「显式切换」判定：只有明确的「再出一份卷 / 做个课件 / 画张导图」等强信号才会跨场景跳转；「题、练习、互动」等弱词不会把教师拽出当前正在打磨的工具。

2.3七大场景工作台

每个工作台共享统一外壳（WorkspaceShell）：左侧对话面板 + 右侧产出/预览面板 + 可拖拽分栏。但右侧产出形态、迭代动作与沉淀方式各具特色。

WS-1找资源

从权威资源库精准检索

右侧产出资源卡片列表（文档 / 视频 / 专辑合集），含匹配度、下载量、页数/题量、三审三校标记、标签。
核心交互追问筛选（「只看实验视频」「整套打包下载」）；每轮检索冻结为一个「轮次」可回看；视频含章节锚点；专辑可展开查看构成。
沉淀资源加入资源篮；下载后归入「我的内容」。
覆盖分类1a / 2a / 3a / 4a 等所有「查找」类。
WS-2出卷子

基于权威题库一键组卷

右侧产出试卷预览（题目、结构、难度梯度、知识点覆盖、答案）。
核心交互多轮命题迭代：「第 5 题换成应用题」「答案放最后」「按某地中考结构」；难度/题量/题型可调。
沉淀导出 Word；存入「我的内容」（试卷）。
覆盖分类3b 生成试卷/习题/变式题。
WS-3做课件

传统 PPT 与互动课件

右侧产出课件预览，支持两种形态：传统 PPT 课件 与 互动课件 · HTML（含课堂活动 / 抢答等交互）。
核心交互先有「配置卡」确认主题/学科/学段/形态，再生成；可「突出情境导入」「加随堂练习」「改成互动课件」。
沉淀存入「我的内容」（PPT 课件 / 互动课件）。
覆盖分类2b 生成PPT · 2c 交互式课件。
WS-4写教案

对齐课标的教学设计成稿

右侧产出教案长文档：三维目标、重难点、教学过程、板书设计等结构化章节。
核心交互先给大纲再成稿；可追加「情境导入」「学情分析」等环节；登录态依据记忆预填版本/学段/学科。
沉淀导出 Word；存入「我的内容」（教案）。
覆盖分类1b 生成教案/教学设计/学案。
WS-5问教材

有据可依的教材问答

右侧产出结构化答案（如「光反应 vs 暗反应」按场所/条件/物质/能量四维对照）+ 教材原文引用（来源、章节、页码、原句）。
核心交互点击答案中的引用标记 → 高亮定位教材原文；支持跨教材/跨版本对比（同一知识点在多版本教材中的编排与侧重差异）；左侧教材目录树可切换章节。
沉淀标注的教材位置 / 答案进入历史对话与内容。
覆盖分类6 学科知识问答（含跨版本对比）。
WS-6画导图

知识结构可视化

右侧产出可视思维导图（章节知识点树形结构）。
核心交互节点可增删改；按目录/考点重组；导出。
沉淀存入「我的内容」（思维导图）。
覆盖分类5 思维导图生成。
WS-7通用助手

兜底问答 + 二次路由

角色当意图未匹配到任何专用工具时的兜底；也是意图识别首次发生的地方。
核心交互直接对话回答教学问题；若识别到具体需求，可带教师跳转进入对应工作台。
覆盖分类9 教学咨询、13 成绩分析、11 翻译，及一切未命中专用工具的教学问题。

2.4会话与多轮迭代模型

产品的灵魂在于：跨场景仍是「同一位助手、同一段对话」。无论从找资源切到出卷子再切到问教材，左侧对话线索都不断。这由全局会话对象 ChatSession 支撑。

机制作用
统一对话日志（log）所有场景共享一条消息流；切换场景时携带历史，不清空。
场景草稿（scratch）各场景保存自己的实时状态（检索轮次、配置、已答状态…），切走再切回不丢失进度。
轮次冻结（rounds）每次提问→一个被冻结的轮次，只追加不覆盖；历史轮次始终可回看。
产物徽标（artifact chip）AI 回复里挂的产物标签，点击可跨场景重新打开那一轮/那次创作。
清空时机仅「新对话」或回到首页时开启全新会话；场景间切换永远延续。
体验收益教师感知到的是「我在跟一个越来越懂上下文的助手连续对话」，而不是在七个割裂工具间反复重启。

2.5小博士记忆系统

记忆让助手「越用越懂你」。它从教师的创作与收藏中学习，形成可见、可管理、可关闭的画像。

记忆构成

  • 画像摘要 — 一段自然语言总结（如「最近一个月主要准备七年级数学，偏好人教版、难度中等…」）。
  • 结构标签 — 学科 / 学段 / 常用版本 / 偏好难度 / 常做类型。
  • 统计 — 课件、卷子、教案、收藏、下载数量。
  • 离散记忆条目 — 每条带「依据」与开关，如「常用人教版（近 30 天 25 次创作中 23 次选择）」，可逐条关闭。
  • 续作线索 — 最近创作与「上次在读的教材」，支持一键续作。

记忆的应用

记忆主要服务于降低重复输入：进入工作台时预填版本/学段/学科；首页提供「继续上次创作」；意图识别时辅助消歧。记忆透明可控 —— 记忆管理页可查看每条依据并单独关闭；首页记忆面板可收起。

隐私与边界记忆仅在登录态生成与展示；未登录时记忆功能收起，改为引导登录的钩子。AI 内容仅供教研参考，需教师结合实际调整。

2.6账号与权限

✓

登录态

解锁记忆、历史对话、我的内容、资源篮的持久化与跨设备同步；首页展示记忆面板与续作。

○

未登录态

全部七大场景正常可用；记忆面板替换为「登录解锁小博士记忆」钩子；触达沉淀功能时弹出登录引导。

设计取向：先让用户用起来，把登录作为「解锁个性化与沉淀」的增益，而非使用门槛。

PART 03 · 交互 PRD

交互 PRD 说明

本部分给出逐界面的交互规格：触发、状态流转、手势与动效、边界与异常处理，以及贯穿全局的交互规范与响应式策略。每条规格以 UI-x 编号，供设计与研发对齐。

3.1首页与智能输入

UI-1

智能输入框（SmartInput）

布局圆角输入卡；内含多行 textarea + 底部操作行（附件按钮、「AI 自动识别场景」提示、开始按钮）。
自适应高度随输入增高，最多 5 行后内部滚动。
占位示例真实教师提问轮播（约 3.4s 切换）；空值时按 Tab 可一键填入当前示例。
提交Enter 提交；Shift+Enter 换行；空内容时「开始」按钮置灰禁用。
聚焦态边框转品牌色 + 外发光（ring）。
附件回形针按钮添加文件，最多 6 个，以 chip 展示、可单独移除。
提交后清空会话 → 进入通用助手并就地运行意图识别（见 UI-4）。
UI-2

首页三种布局（Tweaks 可切换）

对话优先居中大输入框 + 「老师们在问」示例气泡 + 场景胶囊行；强调「先说需求」。
场景宫格输入框 + 场景卡片均衡宫格（带描述）；强调「先选场景」。
助手人格左侧助手自我介绍 + 记忆卡，右侧输入 + 「我可以帮你」场景列表；强调「人格化助手」。
共性三者都同时提供「直接选场景」与「输入需求→AI 识别」两条路径，并都展示记忆面板 / 登录钩子 + 权威背书条。
UI-3

场景快捷入口 / 示例气泡

点击场景直接进入该场景工作台（携带输入框内已有草稿作为首条诉求），跳过意图识别。
点击示例等价于把示例文本作为输入提交，触发意图识别。
悬停卡片上浮 + 按场景色相投影，箭头微显。
互动课件标记「做课件」入口附「含互动课件 / 互动」徽标。

3.2意图识别动效

UI-4

就地意图识别（InlineIntent）

呈现位置作为通用助手对话里的一条 AI 气泡，逐步点亮四步（非全屏遮挡）。
时序约 0.65s → 1.4s → 2.3s → 3.15s 依次推进；完成后约 3.7s 自动落入目标场景。各步含「打字点」动效与勾选完成态。
步骤可视① 理解需求 ② 提取关键信息（实体标签逐个弹入）③ 检索权威库 ④ 匹配场景（目标卡 + 匹配度数字）。
命中专用工具展示场景卡 + 97% 匹配度 → 「已为你打开 XX，继续准备…」并跳转。
未命中第 4 步显示「未匹配到专用工具 → 由通用助手解答」，留在对话直接作答，不跳转。
即时态(instant)从历史/产物徽标重入时，跳过动画直接呈现「已理解」终态。
减动效遵循 prefers-reduced-motion，弱化或跳过过场。
为什么做成可见过程识别动效不是炫技，而是把「有据可依」与「场景判断」做成可被信任的透明过程 —— 教师看到 AI 提取了哪些实体、检索了权威库，才会信任随后的产出。

3.3工作台分栏框架

UI-5

左右分栏与拖拽（桌面端）

结构左对话面板（ChatPanel）+ 中缝拖拽手柄（ws-resizer）+ 右产出面板。
拖拽悬停/拖拽时中缝出现品牌色指示与抓手（grip）；按比例调整左右宽度，col-resize 光标。
顶部工作台头部含返回首页、当前场景标识、场景切换、资源篮入口（带角标）。
对话面板消息流 + 输入框 + 场景化建议胶囊（如「只看实验视频 / 整套打包下载」）。
UI-6

移动端结果抽屉（Sheet）

布局转换窄屏下取消左右分栏；对话占主屏，产出收入可上滑的底部 Sheet（带标签如「资源 / 结果」与图标）。
首轮策略首轮不自动覆盖对话（sheetAnchor 初始为空），避免一进来就被产出挡住。
导航顶部汉堡呼出左侧抽屉导航；右上「新对话」常驻。

3.4各场景关键交互

UI-7找资源

资源卡片与追问筛选

资源卡标题、类型、学段/学科/版本、难度、匹配度、下载量、页数/题量、三审三校标记、标签。
追问筛选自然语言收窄（「只看实验视频」「整套打包」）；每次检索生成新轮次，旧轮次保留可回看（轮次切换器）。
视频资源含时长、清晰度、播放量与章节锚点（可定位到某时间点的知识段）。
专辑/合集展示构成（课件/教案/试卷/习题数量）与条目清单，可展开。
收藏「加入资源篮」即时反馈 toast；重复加入去重提示。
UI-8出卷子 写教案 做课件

生成类：配置 → 生成 → 迭代

配置确认做课件先弹「配置卡」确认主题/学科/学段/形态（PPT vs 互动）；登录态由记忆预填默认值。
生成中骨架屏 / 进度提示；产出在右侧渐次呈现。
迭代动作建议胶囊给出高频改法（出卷子：换题/调难度；教案：加情境导入；课件：加随堂练习/改互动）。
轮次每轮迭代冻结为可回看版本；产物徽标可跨场景重开。
导出/沉淀导出 Word / 保存；自动进入「我的内容」对应类型与状态。
UI-9问教材

引用溯源与跨版本对比

结构化答案按维度对照呈现（如场所/条件/物质/能量），降低阅读负担。
引用溯源答案逐条挂引用标记；点击 → 展开/高亮对应教材原句（来源 + 章节 + 页码）。
教材目录树左侧可切换版本/章节/小节；切换教材不清空对话，仅冷启动才重置问候。
跨版本对比「这个知识点在哪些教材出现过」→ 多版本横向对比（深度、侧重、实验、落点差异）。
UI-10画导图

导图节点操作

生成按章节/考点生成树形结构，分支渐次展开动效。
节点编辑增 / 删 / 改节点；按目录或考点重组。
沉淀导出；存入「我的内容」（思维导图）。

3.5记忆与账号交互

UI-11

记忆面板与记忆管理

首页记忆面板画像摘要 + 标签 + 统计 + 「继续上次创作」；可收起为细条（状态持久化），收起后提示「仍在为你记忆，可在左侧『我的记忆』查看」。
记忆管理页逐条记忆 + 其形成依据 + 独立开关；可关闭单条记忆。
续作点击最近创作/对话 → 在完成态重开对应工作台继续。
未登录记忆面板 → 「登录解锁小博士记忆」钩子（列出可解锁能力），标注「未登录也可正常使用全部场景」。
UI-12

登录与权限引导

触发登录弹窗未登录态点击需登录功能（记忆/内容/历史/资源篮）→ 弹出登录模态，不打断当前页面。
登录后就地解锁，回到原操作；记忆面板、历史列表、账号菜单显现。
登出账号菜单登出 → 回首页并切换为未登录态视图。

3.6全局交互规范

⌨

键盘

Enter 提交 / Shift+Enter 换行 / Tab 填入示例。输入框聚焦态有清晰 ring。

⏳

加载态

骨架屏（skelPulse 脉冲）、打字点（bobDot）、识别步骤逐点亮，避免空白等待。

💬

轻反馈

加入资源篮 / 复制 / 导出等以 toast 短反馈（约 2.2s 自动消失）。

🫙

空状态

未登录历史/内容给出引导文案与登录入口；冷启动工作台给出场景化建议胶囊。

🎞

动效语言

统一缓动 cubic-bezier(.22,1,.36,1)；fadeUp / popIn / drawerIn 等克制入场；尊重减动效偏好。

💡

提示气泡

图标按钮用 data-tip 悬停气泡（约 0.3s 延迟），降低界面文字密度。

免责与边界所有页面统一传达：「AI 内容仅供教研参考，请结合实际教学进行调整」；非教学/噪声类输入由识别引擎引导拦截，不强行承接。

问题分类体系 · 完整正反例

本附录展开 13 大类、23 个末级分类的完整定义、特征词、正例与反例（含易混淆项的归类指向），作为意图识别引擎的标注规范与边界判定依据。数据基线：5 月全量真实提问（60,802 行）抽样分析。

1 · 教案 / 教学设计 / 学案
1a
查找教案 / 教学设计 / 学案
找资源

搜索或获取已存在的教案、教学设计、学案、说课稿、导学案等，期望从资源库检索。

特征词：教案、教学设计、学案、导学案、说课稿、学习任务单、帮我找/搜索/查找 + 教案

正例
  • 初中音乐人音版 乘着歌声的翅膀 教案
  • 高考 最后一课 数学教案
  • 初中化学沪教版 整理与归纳 教案
反例
  • 生成…第18课的教学设计 →1b（含生成动词）
  • 检查这份公开课设计逻辑是否清晰 →9（咨询）
1b
生成教案 / 教学设计 / 学案
写教案

要求 AI 创作新的教案、教学设计、学案、说课稿、导学案。

特征词：生成/写/帮我写/设计 + 教案|教学设计|学案；教学目标/教学过程/板书设计（结构词同现）

正例
  • 生成部编版历史八下第18课 教学设计
  • 写一篇高中数学北师大版 对数的运算 教学设计
  • 根据导学方案生成导学案（物理·力·八年级）
反例
  • 初中化学沪教版整理与归纳教案 →1a（无动词）
  • 生成高三函数一轮复习课件 →2b（课件）
2 · 课件 / PPT（含交互式课件）
2a
查找课件 / PPT
找资源

搜索或获取已存在的课件、PPT、幻灯片等展示文件。

特征词：帮我找/搜索/查找 + 课件|PPT；有没有…课件；免费课件

正例
  • 帮我搜索 沁园春雪 的课件
  • 找一份鲁教版高中地理 热力环流 复习课件
  • 高考阅读理解A篇应用文课件
反例
  • 生成高三函数一轮复习课件 →2b
  • 有「帮我做」语境的「课件」 →2b
2b
生成课件 / PPT（传统幻灯）
做课件

要求 AI 生成课件、PPT、幻灯片内容或大纲，通常指定学科、主题、页数。

特征词：生成/帮我做/制作 + 课件|PPT；逐页描述（封面页/P1/P2…）

正例
  • 生成一个高三函数一轮复习公开课课件
  • 生成中考作文赏析方法总结课件，背景浅蓝
  • 《密度》PPT大纲细化版 + 逐页内容
反例
  • 帮我搜索 沁园春雪 的课件 →2a
  • 根据导学方案生成教学设计 →1b
2c
生成交互式课件（HTML / 代码）
做课件

以 HTML/代码实现的交互式课堂内容：物理仿真、数学动画、知识点演示工具。代码是手段，课堂呈现是本质。

特征词：html + 教学|动画|仿真|互动；以html形式输出 + 教学主题；交互演示

正例
  • 生成html：初中数学勾股定理互动教学动画
  • 空中飞椅项目制作成动画，html 输出
  • 金属球电场线分布三维动画仿真，HTML 输出
反例
  • 生成一个台球小游戏 →10f（娱乐）
  • 网页版计算器（无教学主题）→通用工具
3 · 试卷 / 试题 / 习题
3a
查找试卷 / 真题 / 考卷
找资源

搜索已有的试卷、历年真题、模拟卷，通常指定地区、年份、科目。

特征词：查找/帮我找/搜索 + 试卷|真题|考卷；近X年真题；XX省/市 + 试卷；模考卷

正例
  • 查找北京市2025下学期高一数学期末考卷
  • 找一下2024年全国甲卷
  • 2025年济宁市高中招生物理试题
反例
  • 出一套八年级历史单元测试卷 →3b
  • 2026高考地理冲刺讲义 →4b
3b
生成试卷 / 习题 / 变式题
出卷子

要求 AI 原创题目、习题集、完整试卷或变式题，通常指定题型、数量、难度、知识点。

特征词：出一份/出一套/帮我出/生成/命制 + 试卷|题目|习题；变式题；组卷；指定数量+题型；难度系数

正例
  • 出一份高考化学一轮资料含15道近4年高考题
  • 命制2026四川高考物理适应性训练题（难度0.45）
  • 八上生物选择题30道，答案写最下面
反例
  • 查找北京2025高一数学期末考卷 →3a
  • 搜索斜率之积为-1的题 →3a
3c
试卷 / 命题分析
未命中 → 通用助手

分析已有试卷/一批题目的命题规律、考点分布、难度结构 ——「题目怎么出的」，而非解题、查找或生成。

特征词：命题规律/趋势、考点分布、考情分析、近X年命题趋势、各省命题风格

正例
  • 分析近5年高考物理命题规律及各知识点占比
  • 2025广东中考数学与2024有何不同、新增考点
  • 这套试卷题型结构与难度分布合理吗
反例
  • 1/4圆弧轨道…求… →7（解题）
  • 查找2024全国甲卷 →3a
4 · 知识点 / 讲义 / 复习资料
4a
查找知识点 / 讲义 / 复习资料
找资源

搜索已有的知识点总结、讲义、复习清单、知识梳理，期望获取现成文档。

特征词：查找/找/搜索 + 知识点|讲义|清单；资源名（无生成动词）

正例
  • 找圆锥曲线历届高考真题（含考情分析）
  • 高中数学概率统计知识点
  • 物理知识清单 中考
反例
  • 生成北师大三下知识点清单 →4b
  • 整理2026高考英语听力词语 →4b
4b
生成知识点 / 讲义 / 复习资料
长文本 / 通用

要求 AI 生成知识点清单、专题讲义、复习资料、考前冲刺材料等原创内容。

特征词：必须含创作动词（生成/写/整理/制作/请出）+ 知识点|讲义|总结|清单。纯资源名默认归 4a。

正例
  • 把高中物理学史整理成 word
  • 生成七年级下册期末考前冲刺清单
  • 做一份高考化学回归教材知识要点
反例
  • 2026高考地理冲刺讲义（仅资源名）→4a
  • 物理知识清单中考（短关键词）→4a

6
学科知识问答
问教材 / 通用

直接提出学科知识问题，期待解释概念、分析原理、给出说明；不提供具体题目。

特征词：是什么/为什么/怎么理解/有什么区别 + 学科术语；考情规律总结

正例
  • 悬浊液乳浊液区别
  • 分泌蛋白与内在蛋白合成起点都是核糖体吗
  • 近三年化学试剂保存考查规律，三句总结
反例
  • 总结人工智能发展历程 →10c
  • 1/4圆弧轨道 R=0.2m…求 →7
7
题目求解 / 解析
未命中 → 通用助手

用户提供完整题目（题干、选项或材料），要求作答、讲解思路或分析命题意图。与 6 的关键区别：给出了具体题目。

特征词：题目+选项(A/B/C/D)；下列分析正确的是；题干+求解；阅读理解原文+题目；完整参数

正例
  • 三则招牌选最好的一则并陈述理由（含①②③）
  • 政治选择题，含完整时政材料
  • 1/4圆弧轨道 R=0.2m，传送带 L=5m…求
反例
  • 悬浊液乳浊液区别 →6
  • 出一道高中物理计算题 →3b

教学写作（作文 / 演讲稿 / 文本）
长文本

撰写作文、演讲稿、读后续写、说明文、寄语、报告等文本类内容。

特征词：写一篇/帮我写 + 作文|演讲稿|寄语|报告；续写；作文题+写作要求

特征词：有没有建议；如何让…更好；教学计划；复习计划；备考方案；如何提升

正例
  • 上公开课，有何建议（学生活动/AI 应用）
  • 这篇新闻适合改成阅读理解还是语法填空
  • 制定北师大八下从第四单元到期末十周计划
反例
  • 生成高三函数一轮复习课件 →2b

13
成绩 / 教学数据分析
通用助手

提供/描述学生成绩数据，要求生成成绩分析报告、班级质量分析、薄弱点诊断等数据驱动分析。

特征词：成绩分析、质量分析、平均分/及格率/优秀率、班级分析报告、学情分析（含数据）

正例
  • 据期末成绩生成班级质量分析报告
  • 平均72、及格率85%、优秀率23%，分析问题
  • 据成绩表找出各科薄弱学生
反例
  • 生成高三班主任质量分析（无数据）→8a
  • 分析近5年高考命题趋势 →3c
10a–10f
非教学内容（噪声）
引导拦截 · 不主动承接

不适合设场景的输入，由识别引擎引导拦截或交由通用对话处理：

  • 10a 网址/入口导航 — 「百度网址」「Bing 链接」→ 把 AI 当搜索引擎。
  • 10b 个人生活/娱乐/闲聊 — 健康、电子产品、情感等与教育无关。
  • 10c 一般知识查询（非学科） — GDP 排名、AI 发展史、法律问题等。
  • 10d 产品功能询问 — 「你能判卷吗」「组卷网在哪」→ 适合做功能引导。
  • 10e 闲聊/乱输入/续问片段 — 「你好」「B」「怎么写？」脱离上下文无法独立判断。
  • 10f 娱乐游戏生成 — 台球/跑酷等 H5 游戏，与定位不符，不建议主动承接。
